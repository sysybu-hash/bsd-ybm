# תוכנית: דשבורד מהודר (2026), עם תפריט עליון

מסמך זה **מעדכן** את תוכנית הדשבורד: **תפריט עליון**, מראה **עדכני ל־2026**, ו**מהודר** — לא צעקני ולא «קשת צבעים» (המוקאפ v2 נחשב מדי צבעוני; כיוון המימוש המועדף הוא v3).

## 1. כיוון ויזואלי (מהודר, 2026)

| היבט | כיוון |
|------|--------|
| **פלטה** | רקע **לבן־אפור חם** / `#F8F9FA`, גבולות **דקים** `#E8EAED`, טקסט כהה לניגודיות. **אקסנט יחיד** (טורקיז עמוק / slate) לניווט פעיל ולקישורים — לא אריח KPI בצבע שונה לכל מדד. |
| **KPI** | כרטיסים **אחידים** (לבן, מסגרת 1px, צל עדין או בלי), אייקונים **מונוכרום / duotone**, מספרים בולטים בטיפוגרפיה נקייה. |
| **מרווח ורשת** | הרבה **Whitespace**, רשת 8px, תחושה של מוצרי **Linear / Notion / Vercel** — שקט, מקצועי. |

## 2. תפריט עליון (Top navigation)

**מטרה:** ניווט ראשי **אופקי** בולט, בסגנון דשבורד מודרני (SaaS), ב־RTL.

**אפשרויות ארכיטקטוניות (לבחירה במימוש):**

1. **מומלץ לשלב ראשון:** פס ניווט עליון **בתוך אזור התוכן הראשי** (מתחת ל־command palette / שורת כלים אם קיימת), מיישר את אותם **מזהי נתיבים** כמו `buildAppNavCollection` / [`app-nav.ts`](components/app-shell/app-nav.ts) — כלומר **אותן יעדים כמו בסרגל הצד**, בלי לשכפל לוגיקת הרשאות (שימוש באותם בוני ניווט + `hiddenPrimaryRouteIds`).

2. **הרחבת Shell (שלב 2):** הוספת רכיב משותף `WorkspaceTopNav` ב־[`AppShellV2.tsx`](components/app-shell/AppShellV2.tsx) כך שכל עמודי `/app` יקבלו תפריט עליון עקבי; הסרגל הצד נשאר ל־**משני / mobile** או מצומצם (תלוי החלטת UX).

**התנהגות רספונסיבית:** במובייל — תפריט עליון כגלילה אופקית (כבר קיים דפוס דומה ב־`MobilePill`); לוודא שלא נוצר כפל מביך עם ה־sidebar.
